# Oh My PPTX

Use this skill when the user has an HTML slide design and wants a native editable `.pptx`.

## Requirements

- Node.js 18 or newer.
- Latest `deckops` CLI with HTML-to-PPTX batch conversion.
- DeckFlow authentication via `deckops login` or configured token/space.

Install or upgrade if needed:

```bash
npm install -g deckops@latest
deckops login
```

## Standard Workflow

1. Create or locate the source `.html`.
2. Inspect the split output:

```bash
node /path/to/oh-my-pptx/scripts/html-to-pptx.mjs input.html --inspect-only --keep-temp --json
```

3. Check `slideCount`, `splitMode`, `canvas`, and `warnings`.
4. If inspection reports relative asset references such as `assets/...`, pause and fix them before uploading. Relative paths can succeed locally but still fail in DeckFlow if the generated HTML pages do not carry the referenced files with them.
5. Open the kept `slide-*.html` files if layout looks risky.
6. Convert:

```bash
node /path/to/oh-my-pptx/scripts/html-to-pptx.mjs input.html --out output.pptx --timeout 600
```

Return only the final PPTX unless the user asks for debug files.

## How The Wrapper Prepares HTML

- Finds common slide containers such as `.slide`, `section.slide`, `[data-slide]`, and `[data-page]`.
- Preserves the original DOM for selector-based splitting and hides non-active slides.
- Adds `visible` to the active slide so reveal-style decks render their final state.
- Inlines local CSS, images, and fonts when possible.
- Strips scripts by default; use `--script-mode keep` only when slide content is generated by JavaScript.
- Detects fixed dimensions or responsive `100vw`/`100vh` slide canvases and passes size to DeckFlow.
- Warns when the generated pages still contain relative asset references. Treat that as a preflight failure unless you intentionally plan to upload those files together.

DeckFlow receives the generated single-page HTML files in order:

```bash
deckops --json convert slide-001.html slide-002.html --to pptx --width 1920 --height 1080 --need-embed-fonts false
```

## Options To Reach For

- `--canvas 1920x1080` or `--width 1920 --height 1080` when the canvas is ambiguous.
- `--split-mode scroll --page-count N` for long responsive pages that are not already slide-based.
- `--inline-assets false` only when you intentionally want to keep a sibling `assets/` directory next to every generated page and you have verified DeckFlow can access it.
- `--need-embed-fonts true` only when the user asks for embedded fonts.
- `--strict` when warnings should stop conversion.

## Troubleshooting

- Auth errors: run `deckops login`.
- Missing batch conversion options: upgrade with `npm install -g deckops@latest`.
- Relative asset warnings such as `assets/foo.css` or `assets/bar.png`: fix the HTML packaging before conversion, or inline the assets. Do not keep retrying remote converts until inspection is clean.
- Missing JS-rendered content: rerun with `--script-mode keep`, or pre-render the content into static HTML.
- Layout drift: inspect the kept split HTML files and prefer explicit slide containers or explicit canvas dimensions.
